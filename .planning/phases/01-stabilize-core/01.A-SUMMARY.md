# Phase 1 Plan 01.A: STEP Upload Hardening Summary

One-liner: Closed the STEP temp-file leak (CORE-01) and added pre-parse magic-byte + triangle-cap defense-in-depth (CORE-07) so the /validate endpoint cannot be weaponized into a disk-fill or RAM-exhaustion DoS before Phase 2 opens it to the public.

## Files Changed

Created:
- `backend/src/api/upload_validation.py` — exports `_STEP_MAGIC`, `validate_magic`, `enforce_triangle_cap`, `_max_triangles`.
- `backend/tests/test_step_parser.py` — cleanup + 0o600 regression tests.
- `backend/tests/test_upload_validation.py` — magic-byte + triangle-cap tests.

Modified:
- `backend/src/parsers/step_parser.py` — added `import os`; rewrote `parse_step_from_bytes` to use try/finally with `os.chmod(tmp.name, 0o600)` before write and guaranteed `os.unlink` (swallowing `FileNotFoundError`) in the finally clause. Public signatures (`is_step_supported`, `parse_step`, `parse_step_from_bytes`) unchanged.
- `backend/src/api/routes.py` — added `from src.api.upload_validation import enforce_triangle_cap, validate_magic`; `_parse_mesh` now calls `validate_magic(data, suffix)` before parser dispatch and `enforce_triangle_cap(mesh)` after each parse before return.
