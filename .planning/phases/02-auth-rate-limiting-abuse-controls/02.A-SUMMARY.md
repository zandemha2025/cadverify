# Phase 02 Plan 02.A: Signup + Key Issuance Summary

**One-liner:** Google OAuth + Resend magic-link signup mint `cv_live_<prefix>_<secret>` API keys backed by Argon2id hashes and an HMAC-indexed lookup table, gated by Cloudflare Turnstile and a 20-domain hard-reject disposable-email list with a D-11 allowlist override for Proton/Tuta/Fastmail.

## What Shipped

- **Alembic scaffold** — `alembic.ini` + async `env.py` + migration `0001_create_users_api_keys` creating both tables with `ix_api_keys_user_id` and unique `ix_api_keys_hmac_index`.
- **`backend/src/auth/` package** (8 files):
  - `hashing.py` — `mint_token()` returns (`cv_live_<8b62>_<32b62>`, prefix, Argon2id(t=3,m=64MB,p=4) hash); `hmac_index()` compat stub using `API_KEY_PEPPER`.
  - `disposable.py` — 20-domain hard-reject frozenset, 12-domain never-block allowlist that wins over every other list, `normalize_email` collapsing gmail dots + stripping `+tags`.
  - `turnstile.py` — `verify_turnstile` fails closed on network error / non-200 / success:false.
  - `models.py` — async SQLAlchemy engine + `upsert_user`, `create_api_key`, `lookup_api_key`, `touch_last_used` helpers.
  - `magic_link.py` — `/magic/start` (Turnstile → classify → Resend send) and `/magic/verify` (HMAC decode → Redis `GETDEL` → mint key → 303 + `cv_mint_once` cookie).
  - `oauth.py` — authlib Google register + `/google/start` + `/google/callback` mirroring the magic-link mint/redirect flow.
- **`backend/main.py`** — added `SessionMiddleware` (authlib state storage) + `/auth` router mounts for oauth + magic.
- **Frontend auth routes** — `(auth)/signup/page.tsx` (Google anchor + email form + `<div class="cf-turnstile">`), `(auth)/signup/actions.ts` server action, `(auth)/magic/verify/page.tsx` server component that proxies the backend 303.

## Tests (17 passing)

- `test_auth_turnstile.py` — 3 tests (respx-mocked): success, `success:false`, non-200 upstream.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Frontend server action signature type error**
- **Found during:** Task A8 (tsc verification).
- **Issue:** `startMagic` returned `{ok: true}`, but Next 16 `<form action={...}>` requires `(fd: FormData) => void | Promise<void>`.
- **Fix:** Changed return type to `Promise<void>` and dropped the trailing return.
- **File modified:** `frontend/src/app/(auth)/signup/actions.ts`.
- **Commit:** `f17c862`.

**2. [Rule 2 - Missing] Added `itsdangerous` to requirements.txt**
- **Found during:** Task A7 (main.py wiring).
- **Issue:** `starlette.middleware.sessions.SessionMiddleware` requires `itsdangerous` at runtime; plan listed `authlib` but not this transitive.
- **Fix:** Added `itsdangerous>=2.2.0` to `backend/requirements.txt`.
- **Commit:** `df414f6` (initial dep commit).

**3. [Rule 1 - Bug] `_base62_chunk` could underfill**
- **Found during:** Task A2 review.
- **Issue:** `secrets.token_urlsafe(n).replace(...)` can drop characters and return a string shorter than `out_len`, meaning `mint_token` could yield a <8 or <32 char chunk on rare inputs.
- **Fix:** Pad with `secrets.choice(alphanum)` until target length is met.
- **Commit:** `3687590`.
