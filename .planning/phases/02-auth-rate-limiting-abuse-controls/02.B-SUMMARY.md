## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Pydantic PEP 604 union incompatibility with Python 3.9**
- Found during: Task B3 verification (`from src.auth.keys_api import router` crashed).
- Issue: `last_used_at: str | None` in `KeyOut` raised `TypeError: unsupported operand type(s) for |: 'type' and 'NoneType'` because Pydantic evaluates field annotations at runtime even with `from __future__ import annotations`, and this project runs Python 3.9.
- Fix: Switched to `Optional[str] = None` from `typing`.
- Files: `backend/src/auth/keys_api.py`
- Commit: `161a180`

**2. [Rule 1 - Bug] Dependency override signature mismatch in test**
- Found during: Task B4 (first run — all 6 tests 422).
- Issue: `fake_session(request)` without a type annotation caused FastAPI to interpret `request` as a body field, returning 422 for every call.
- Fix: Changed to `async def fake_session() -> int` (no params).
- Files: `backend/tests/test_keys_api.py`
- Commit: folded into `940ff70`

**3. [Rule 2 - Defensive] Widened `verify_token` catch**
- Reason: argon2-cffi can raise unexpected `ValueError`/encoding errors on malformed input. Added a defensive `except Exception: return False` after the known argon2 exceptions to keep the "never raises" contract firm.
- Files: `backend/src/auth/hashing.py`
- Commit: `643fc7d`
