# Phase 02 Plan 02.C: Auth Enforcement, Rate Limiting, Kill-Switch Summary

Wires the `require_api_key` dependency, 60/hr + 500/day slowapi rate limits, and `ACCEPTING_NEW_ANALYSES` kill-switch into every `/api/v1/*` route, with a CI grep job that fails the build on regression (AUTH-06/07/09).

## What Shipped

### New modules
- **`src/auth/require_api_key.py`** — `AuthedUser` pydantic model + `require_api_key` dependency. Parses `Authorization: Bearer cv_live_...`, looks up by `hmac_index`, verifies via argon2 `verify_token`, raises structured `401 {code, message, doc_url}` on any failure (same `auth_invalid` code for unknown + revoked → no user enumeration). Stores user on `request.state.authed_user` so the rate-limit `key_func` can read it. `touch_last_used` runs fire-and-forget via `asyncio.create_task`.
- **`src/auth/rate_limit.py`** — slowapi `Limiter` keyed by `api_key_id` (IP fallback for public routes), `headers_enabled=True`, `in_memory_fallback_enabled=True` (fails open to local memory if Redis dies), `storage_uri=$REDIS_URL` or `memory://`. `rate_limit_handler` emits 429 with `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset` and structured body.
- **`src/auth/kill_switch.py`** — `is_accepting()` with a 30-s threading-locked in-process cache. False for case-insensitive `false/0/no/off`; default True. `require_kill_switch_open` dependency raises `503 + Retry-After: 3600` with `service_paused` code.

### routes.py modifications (6 routes updated, 0 skipped)
All routes under `/api/v1/` (`router` prefix in `main.py`) now carry `Depends(require_api_key) + @limiter.limit("60/hour;500/day")`:

| Route | Method | Auth | Rate Limit | Kill-Switch |
|------|------|------|------|------|
| `/validate` | POST | ✓ | ✓ | ✓ |

Public routes kept exempt (mounted under different prefixes, confirmed not in `routes.py`):

- `/auth/google/*`, `/auth/magic/*` (oauth.py, magic_link.py)
- `/api/v1/keys/*` (keys_api.py — gated by `require_dashboard_session` instead)

### main.py
Registered `app.state.limiter = limiter`, `RateLimitExceeded` handler, and `SlowAPIMiddleware` before routers are included.

### CI: `.github/workflows/ci.yml::route-auth-coverage`
New job runs an inline Python grep across `backend/src/api/routes.py`: every `@router.{verb}(...)` whose path isn't `/health`, `/auth/*`, or `/s/*` must reference `require_api_key` within the next 1500 characters. Fails the build on regression.

### scripts/ops/kill-switch.sh
`./scripts/ops/kill-switch.sh {on|off}` → `fly secrets set ACCEPTING_NEW_ANALYSES=... && fly deploy -a cadverify-api`. Exits 1 on missing/invalid arg. `chmod +x` set.

### conftest.py auth bypass (deviation)
Pre-02.C tests (`test_api.py`, `test_rule_packs.py`, `test_frontend_errors.py`, `test_step_corruption.py`, `test_large_mesh.py`) call `/api/v1/*` directly without Bearer headers. Added `_bypass_api_key_auth` autouse fixture that wraps `importlib.reload` so each `main`-reload during a test's `client` fixture re-installs `app.dependency_overrides[require_api_key] = fake_user`. Tests exercising real auth build their own isolated FastAPI app instances and are unaffected.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] slowapi 0.1.9 requires `response: Response` kwarg for header injection**
- Found during: Task C4
- Issue: With `headers_enabled=True` and `SlowAPIMiddleware`, slowapi's `__limit_decorator` still calls `_inject_headers(kwargs.get("response"), ...)` after each call. When routes returned `dict` without a `response: Response` parameter, `None` was passed, raising `Exception("parameter response must be an instance of starlette.responses.Response")`.
- Fix: Added `response: Response` kwarg to every rate-limited route in `routes.py`. FastAPI injects a fresh response object each request; slowapi writes headers into it.
- Files modified: `backend/src/api/routes.py`
- Commit: `e725313`

**2. [Rule 3 - Blocking] Pre-existing tests fail with 401 after wiring auth**
- Found during: Task C4
- Issue: 7 tests in `test_api.py`, plus tests in `test_rule_packs.py`, `test_frontend_errors.py`, `test_step_corruption.py`, `test_large_mesh.py`, call `/api/v1/validate`, `/api/v1/processes`, etc. directly without a Bearer header. Wiring `require_api_key` breaks all of them.
- Fix: Added `_bypass_api_key_auth` autouse fixture in `conftest.py` that wraps `importlib.reload` so any test-local `client` fixture that reloads `main` gets `app.dependency_overrides[require_api_key] = fake_user` re-applied on the fresh app instance. Real-auth tests build isolated apps and are unaffected.
- Files modified: `backend/tests/conftest.py`
- Commit: `e725313`

**3. [Rule 3 - Blocking] slowapi not in requirements.txt**
- Found during: Task C2
- Issue: `src/auth/rate_limit.py` imports `slowapi` but the package isn't listed in `backend/requirements.txt`. Deploy would fail.
- Fix: Added `slowapi>=0.1.9`.
- Files modified: `backend/requirements.txt`
- Commit: `1801773`
