# Phase 02 Plan 02.D: Signup Limits + Log Scrubbing + CORS Hardening + Dashboard Session Summary

AUTH-08/10/11 delivered: signup is now rate-limited at the IP and normalized-email
layers (per-IP 3/hr, per-email 1/24h → 1/7d on disposable soft-flag), `cv_live_*`
tokens and `Authorization` headers are scrubbed in structlog + Sentry (double
defense + CI grep guard), CORS is locked down to a regex origin with explicit
headers and `allow_credentials=False`, and the dashboard session cookie now has
a full HMAC-signed 30-day rolling lifecycle wired into both OAuth and magic-link
success paths.

## Deviations from Plan

None — plan executed as written. Minor test assertion in `test_sentry_leak.py`
and `test_auth_scrubbing.py` adjusted to explicitly strip the redacted marker
`cv_live_***REDACTED***` before asserting no `cv_live_` remains (the marker
itself contains the literal substring, so a naive `"cv_live_" not in dump`
check would fail). CI grep pattern (`cv_live_[A-Za-z0-9_]+`) does NOT match
the redacted marker because `*` is not in `[A-Za-z0-9_]`, so CI is correct
as-is.
