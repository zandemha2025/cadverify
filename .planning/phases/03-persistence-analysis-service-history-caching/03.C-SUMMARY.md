## What Was Built

- **GET /api/v1/analyses**: Cursor-paginated list of user's analyses. Supports `cursor` (ULID), `limit` (1-100, default 20), `verdict` (pass/issues/fail) query params. Returns summary fields: id, filename, file_type, verdict, face_count, duration_ms, created_at, process_count, best_process. Response includes `next_cursor` and `has_more` for pagination.

- **GET /api/v1/analyses/{analysis_id}**: Full analysis detail by ULID. Returns metadata envelope with id, filename, file_type, created_at, and the complete `result_json`. Returns 404 for analyses belonging to other users or nonexistent ULIDs.

Both endpoints enforce per-user isolation (user_id from require_api_key) and rate limiting (60/hour, 500/day).

## Threat Mitigations Applied

- **T-03C-01 (IDOR)**: Both queries filter by `user_id == user.user_id` from authenticated session. 404 returned for other users' analyses.
- **T-03C-02 (Cursor manipulation)**: Invalid ULID cursors produce empty result sets via safe `<` comparison.
- **T-03C-03 (N+1 queries)**: List endpoint uses denormalized columns only, single query per request.
