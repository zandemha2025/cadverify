<objective>
Create a two-tier mesh repair service (trimesh.repair pre-pass + pymeshfix.MeshFix fallback) exposed at `POST /api/v1/validate/repair`. The endpoint accepts STL/STEP file uploads, attempts repair, runs a full re-analysis on the repaired mesh via `analysis_service.run_analysis()`, and returns both original and repaired analyses plus base64-encoded repaired STL bytes. Timeout-bounded at 30s with graceful fallback to original analysis on failure.
</objective>

<must_haves>
- A non-watertight STL sent to POST /api/v1/validate/repair returns repaired STL bytes plus a full re-analysis
- A previously-repaired hash hits cache and skips pymeshfix re-run (via analysis_service dedup)
- pymeshfix invocation is timeout-bounded and falls back cleanly on failure (no worker hangs)
- Face-count cap at 500k rejects oversize meshes with 413
</must_haves>

<threat_model>
## Threat Model (ASVS L1)

| ID | Threat | Severity | Mitigation | Verify |
|----|--------|----------|------------|--------|
| T-05A-01 | DoS via pathological mesh causing pymeshfix hang | HIGH | REPAIR_TIMEOUT_SEC (30s default) via asyncio.wait_for; face-count cap at 500k | Test: mesh >500k returns 413; timeout returns graceful fallback |
| T-05A-02 | Memory exhaustion from large mesh repair | HIGH | REPAIR_MAX_FACES=500000 rejects before repair; pymeshfix memory ~1 byte/face | Test: 500k cap enforced; no OOM in normal path |
| T-05A-03 | File upload abuse (oversized/malformed) | MEDIUM | Reuse existing _read_capped (MAX_UPLOAD_MB), validate_magic, enforce_triangle_cap | Test: existing upload validation guards active on repair endpoint |
| T-05A-04 | Base64 response size amplification | LOW | Max repaired STL ~50 MB -> ~67 MB b64; acceptable at beta scale; face-count cap limits practical size | Accept: documented limit |
| T-05A-05 | pymeshfix segfault crashes worker | MEDIUM | In-process execution with timeout; Fly auto-restarts; face-count cap reduces probability | Accept: beta-scale risk; subprocess isolation deferred |
</threat_model>

<action>
Create `backend/src/services/repair_service.py` with the following structure:

1. **Imports**: `asyncio`, `base64`, `logging`, `os`, `time`, `trimesh`, `numpy`. Lazy import of `pymeshfix` (wrapped in try/except ImportError).

2. **Config functions** (same lazy-read pattern as routes.py):
   - `_repair_timeout_sec() -> float`: reads `REPAIR_TIMEOUT_SEC` env var, default `30.0`, min `0.1`
   - `_repair_max_faces() -> int`: reads `REPAIR_MAX_FACES` env var, default `500000`, min `1`

3. **Logger**: `logger = logging.getLogger("cadverify.repair_service")`

4. **`_tier1_repair(mesh: trimesh.Trimesh) -> trimesh.Trimesh`** (sync function):
   - `mesh.remove_degenerate_faces()`
   - `trimesh.repair.fix_normals(mesh)`
   - `trimesh.repair.fix_inversion(mesh)`
   - `trimesh.repair.fill_holes(mesh)`
   - `trimesh.repair.fix_winding(mesh)`
   - Return mesh
   - Log: `logger.info("Tier 1 repair: watertight=%s faces=%d", mesh.is_watertight, len(mesh.faces))`

5. **`_tier2_repair(mesh: trimesh.Trimesh) -> trimesh.Trimesh`** (sync function):
   - Try import pymeshfix; if ImportError, log warning and return mesh unchanged
   - `meshfix = pymeshfix.MeshFix(mesh.vertices, mesh.faces)`
   - `meshfix.repair(verbose=False)`
   - Create new trimesh: `repaired = trimesh.Trimesh(vertices=meshfix.v, faces=meshfix.f)`
   - Log: `logger.info("Tier 2 pymeshfix repair: watertight=%s faces=%d", repaired.is_watertight, len(repaired.faces))`
   - Return repaired
   - On `RuntimeError` or `Exception`: log warning, return original mesh unchanged

6. **`_do_repair(mesh: trimesh.Trimesh) -> tuple[trimesh.Trimesh, str, int]`** (sync function):
   - Record original face count: `original_faces = len(mesh.faces)`
   - Run `_tier1_repair(mesh)`
   - If `mesh.is_watertight`: return `(mesh, "trimesh", 0)` — tier, holes_filled placeholder
   - Run `_tier2_repair(mesh)` -> `repaired`
   - Count holes_filled as `abs(len(repaired.faces) - original_faces)` (approximate)
   - Return `(repaired, "pymeshfix", holes_filled)`

7. **`async def repair_mesh(file_bytes, filename, processes, rule_pack, user, session) -> dict`** (main entry point):
   - Import lazily from routes: `_parse_mesh`, `_read_capped` (not needed — already have bytes), `validate_magic`, `enforce_triangle_cap` already called by the route handler before this function
   - Import `analysis_service`
   - Parse mesh: call `_parse_mesh(file_bytes, filename)` -> `(mesh, suffix)` (imported lazily from routes to avoid circular)
   - **Face-count guard**: if `len(mesh.faces) > _repair_max_faces()`: raise `HTTPException(413, detail=f"Mesh has {len(mesh.faces):,} faces, exceeds REPAIR_MAX_FACES limit of {_repair_max_faces():,}. Simplify the mesh before attempting repair.")`
   - Run original analysis first: `original_result = await analysis_service.run_analysis(file_bytes, filename, processes, rule_pack, user, session)`
   - Start repair timer: `repair_start = time.time()`
   - Run repair with timeout:
     ```python
     loop = asyncio.get_event_loop()
     try:
         repaired_mesh, tier, holes_filled = await asyncio.wait_for(
             loop.run_in_executor(None, _do_repair, mesh),
             timeout=_repair_timeout_sec(),
         )
     except (asyncio.TimeoutError, Exception) as exc:
         logger.warning("Repair failed: %s", exc)
         return {
             "original_analysis": original_result,
             "repair_applied": False,
             "repair_details": {"error": str(exc)},
             "repaired_analysis": None,
             "repaired_file_b64": None,
         }
     ```
   - `repair_duration_ms = round((time.time() - repair_start) * 1000, 1)`
   - Check if repair actually helped: if `not repaired_mesh.is_watertight` and tier was only "trimesh": return with `repair_applied: False`
   - Export repaired mesh to binary STL: `repaired_stl_bytes = repaired_mesh.export(file_type='stl')`
   - Base64 encode: `repaired_b64 = base64.b64encode(repaired_stl_bytes).decode("ascii")`
   - Run re-analysis on repaired mesh bytes: `repaired_result = await analysis_service.run_analysis(repaired_stl_bytes, f"{filename}-repaired.stl", processes, rule_pack, user, session)`
   - Build repair_details dict: `{"tier": tier, "original_faces": original_faces, "repaired_faces": len(repaired_mesh.faces), "holes_filled": holes_filled, "duration_ms": repair_duration_ms}`
   - Return combined response:
     ```python
     {
         "original_analysis": original_result,
         "repair_applied": True,
         "repair_details": repair_details,
         "repaired_analysis": repaired_result,
         "repaired_file_b64": repaired_b64,
     }
     ```
</action>

2. **New endpoint**:
```python
@router.post("/validate/repair", dependencies=[Depends(require_kill_switch_open)])
@limiter.limit("60/hour;500/day")
async def validate_repair(
    request: Request,
    response: Response,
    file: UploadFile = File(...),
    processes: Optional[str] = Query(
        None,
        description="Comma-separated process types for re-analysis. Leave empty for all.",
    ),
    rule_pack: Optional[str] = Query(
        None,
        description="Industry rule pack: aerospace, automotive, oil_gas, medical.",
    ),
    user: AuthedUser = Depends(require_api_key),
    session: AsyncSession = Depends(get_db_session),
):
    """Upload a STEP or STL file, attempt mesh repair, and get before/after analysis."""
    if rule_pack:
        pack = get_rule_pack(rule_pack)
        if pack is None:
            raise HTTPException(
                status_code=400,
                detail=f"Unknown rule pack '{rule_pack}'. Available: {available_rule_packs()}",
            )

    data = await _read_capped(file)
    return await repair_service.repair_mesh(
        file_bytes=data,
        filename=file.filename or "unknown",
        processes=processes,
        rule_pack=rule_pack,
        user=user,
        session=session,
    )
```

Place this between the `validate_quick` endpoint and the `list_rule_packs` endpoint.
