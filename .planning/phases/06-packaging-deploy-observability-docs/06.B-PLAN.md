<objective>
Deliver PKG-03 (docker-compose.yml for local dev), PKG-04 (Fly.io fly.toml with processes
+ volumes), and PKG-05 (managed Postgres config with connection pooling).

The docker-compose stack mirrors production topology: backend (web), worker, Postgres,
Redis, frontend. The fly.toml adds process groups so web and worker run as separate
Fly machines from the same image.
</objective>

<task id="06B-1">
<title>Create docker-compose.yml</title>
<read_first>
- backend/Dockerfile
- frontend/Dockerfile
- backend/requirements.txt
</read_first>
<action>
Create `docker-compose.yml` at repo root:

```yaml
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    env_file: .env
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_started
    environment:
      - PORT=8000

  worker:
    build: ./backend
    command: ["arq", "src.jobs.worker.WorkerSettings"]
    env_file: .env
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_started

  postgres:
    image: postgres:16-alpine
    volumes:
      - pgdata:/var/lib/postgresql/data
    environment:
      POSTGRES_DB: cadverify
      POSTGRES_USER: cadverify
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-localdev}
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U cadverify"]
      interval: 5s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  frontend:
    build: ./frontend
    ports:
      - "3000:3000"
    environment:
      - NEXT_PUBLIC_API_URL=http://localhost:8000/api/v1

volumes:
  pgdata:
```
</action>

<task id="06B-3">
<title>Update backend/fly.toml with processes, mounts, and deploy sections</title>
<read_first>
- backend/fly.toml
- .planning/phases/06-packaging-deploy-observability-docs/06-RESEARCH.md
</read_first>
<action>
Replace `backend/fly.toml` with:

```toml
app = "cadverify-api"
primary_region = "iad"

[build]
  dockerfile = "Dockerfile"

[processes]
  web = "uvicorn main:app --host 0.0.0.0 --port 8000 --workers 2"
  worker = "arq src.jobs.worker.WorkerSettings"

[env]
  LOG_LEVEL = "INFO"
  MAX_UPLOAD_MB = "100"
  SAM3D_ENABLED = "false"
  ACCEPTING_NEW_ANALYSES = "true"
  ANALYSIS_TIMEOUT_SEC = "30"
  BLOB_STORAGE_PATH = "/data/blobs"

[http_service]
  processes = ["web"]
  internal_port = 8000
  force_https = true
  auto_stop_machines = "suspend"
  auto_start_machines = true
  min_machines_running = 1

  [http_service.concurrency]
    type = "requests"
    hard_limit = 50
    soft_limit = 25

[[mounts]]
  source = "cadverify_data"
  destination = "/data"
  processes = ["web"]
  initial_size = "1gb"

[deploy]
  release_command = "sh -c 'DATABASE_URL=$DATABASE_URL_DIRECT alembic upgrade head'"

[[vm]]
  memory = "1gb"
  cpu_kind = "shared"
  cpus = 2
```

Key changes from existing fly.toml:
- Added `[processes]` with web + worker
- `[http_service]` now has `processes = ["web"]` to route HTTP only to web machines
- Changed `min_machines_running` from 0 to 1 (no cold start)
- Changed `auto_stop_machines` from "stop" to "suspend" (faster wake)
- Added `[[mounts]]` for PDF blob cache volume
- Added `[deploy] release_command` for Alembic migrations
- Added `BLOB_STORAGE_PATH` and `ACCEPTING_NEW_ANALYSES` to `[env]`
</action>
