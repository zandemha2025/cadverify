## Decisions Made

1. **5-service topology**: backend, worker, postgres:16-alpine, redis:7-alpine, frontend -- mirrors production.
2. **Worker reuses backend image**: Same build context with `arq` command override (no duplicate image).
3. **Fly processes**: web + worker from single image; HTTP routed only to web via `processes = ["web"]`.
4. **No cold start**: `min_machines_running = 1` ensures always-on.
5. **Suspend over stop**: `auto_stop_machines = "suspend"` for faster wake on idle machines.
6. **Release command for migrations**: `DATABASE_URL=$DATABASE_URL_DIRECT alembic upgrade head` bypasses PgBouncer for DDL (T-06B-03 mitigation).
7. **Placeholder-only secrets in .env.example**: No real credentials (T-06B-01 mitigation).
8. **Non-sensitive env in fly.toml**: Only LOG_LEVEL, MAX_UPLOAD_MB, etc. in [env]; all secrets via `fly secrets set` (T-06B-02 mitigation).
