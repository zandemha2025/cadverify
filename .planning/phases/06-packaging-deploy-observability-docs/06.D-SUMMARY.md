## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] StarletteHTTPException handler missing**
- **Found during:** Task 5 (test_404_structured failing)
- **Issue:** FastAPI's default 404 for unmatched routes raises `starlette.exceptions.HTTPException`, not `fastapi.exceptions.HTTPException`. The structured handler only caught the FastAPI subclass.
- **Fix:** Registered handler for both `HTTPException` and `StarletteHTTPException` in main.py. Changed errors.py to import from starlette.exceptions as the base class.
- **Files modified:** backend/main.py, backend/src/api/errors.py
- **Commit:** 8591d30

**2. [Rule 3 - Blocking] No get_redis helper exists**
- **Found during:** Task 2 (health endpoint implementation)
- **Issue:** Plan referenced `src.auth.rate_limit.get_redis` which does not exist. Redis is managed internally by slowapi's Limiter.
- **Fix:** Health probe creates its own direct connection from `REDIS_URL` env var using `redis.asyncio.from_url()` with a 2-second socket timeout. Reports healthy when `REDIS_URL` is unset (in-memory fallback mode).
- **Files modified:** backend/src/api/health.py
- **Commit:** ae13227

**3. [Rule 3 - Blocking] get_async_engine does not exist**
- **Found during:** Task 2
- **Issue:** Plan referenced `get_async_engine()` but the actual function in src/db/engine.py is `get_engine()`.
- **Fix:** Used `get_engine()` from `src.db.engine`.
- **Files modified:** backend/src/api/health.py
- **Commit:** ae13227
