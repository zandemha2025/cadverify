## Decisions Made

1. **Auth-gated landing vs dashboard:** Landing page shown to unauthenticated visitors (checks localStorage for `cadverify_api_key`). Authenticated users see the full analysis dashboard. This preserves existing functionality.

2. **unstable_retry over reset:** Next.js 16 introduced `unstable_retry` which re-fetches and re-renders. Used this as primary recovery mechanism per framework docs.

3. **Sentry disabled by default:** `enabled: !!process.env.NEXT_PUBLIC_SENTRY_DSN` ensures no Sentry traffic when DSN is not configured.
