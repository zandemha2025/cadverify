## What Was Built

1. **job_service.py** -- Job orchestration layer with `create_sam3d_job` (idempotent by analysis_id+sam3d), `get_job_for_user` (auth-scoped), and `save_mesh_blob` (idempotent by hash).

2. **Async submit path** -- `POST /api/v1/validate?segmentation=sam3d` runs sync analysis first, persists it, saves mesh blob for worker, creates idempotent job row, enqueues to arq, returns HTTP 202 with `{analysis_id, job_id, poll_url, result}`.

3. **Polling endpoints** -- `GET /api/v1/jobs/{id}` returns status/timestamps/result_url. `GET /api/v1/jobs/{id}/result` returns full SAM-3D result when done/partial. Both auth-protected; returns 404 (not 403) for other users' jobs per D-12.

4. **10 tests** covering status polling (queued, done), auth scoping (404 for other user), result retrieval (not ready, done, partial), idempotent job creation, and idempotent blob storage.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] SQLAlchemy Job.__new__ fails in tests**
- **Found during:** Task 4
- **Issue:** `Job.__new__(Job)` bypasses SQLAlchemy's init, causing `AttributeError` on instrumented attributes.
- **Fix:** Used `MagicMock(spec=Job)` instead for test job objects.
- **Commit:** af3427a

**2. [Rule 2 - Missing functionality] get_latest_analysis_id helper**
- **Found during:** Task 2
- **Issue:** Plan suggested refactoring `run_analysis` return type or adding `get_last_analysis_id`. Adding a return-type change would break all callers.
- **Fix:** Added `get_latest_analysis_id(session, user_id, mesh_hash)` to analysis_service.py -- queries by user+mesh_hash with descending created_at. Minimal surface change.
- **Commit:** a96265c
