## Task 1: Create .github/dependabot.yml

<type>execute</type>

<read_first>
- .github/workflows/ (verify .github directory exists for CI workflows)
</read_first>

<files>
- .github/dependabot.yml
</files>

<action>
Create `.github/dependabot.yml`:
```yaml
# Dependabot configuration — weekly dependency updates for both ecosystems.
# Docs: https://docs.github.com/en/code-security/dependabot/dependabot-version-updates/configuration-options-for-the-dependabot.yml-file
version: 2
updates:
  # Backend — pip (requirements files in /backend)
  - package-ecosystem: "pip"
    directory: "/backend"
    schedule:
      interval: "weekly"
      day: "monday"
    open-pull-requests-limit: 5
    labels:
      - "dependencies"
      - "backend"
    commit-message:
      prefix: "chore(deps)"

  # Frontend — npm (package.json in /frontend)
  - package-ecosystem: "npm"
    directory: "/frontend"
    schedule:
      interval: "weekly"
      day: "monday"
    open-pull-requests-limit: 5
    labels:
      - "dependencies"
      - "frontend"
    commit-message:
      prefix: "chore(deps)"

  # GitHub Actions — keep CI workflows up to date
  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
    open-pull-requests-limit: 3
    labels:
      - "dependencies"
      - "ci"
    commit-message:
      prefix: "chore(ci)"
```
</action>

<verify>
Run: `cat .github/dependabot.yml | head -5` — should show `version: 2`.
Run: `grep "package-ecosystem" .github/dependabot.yml | wc -l` — should show 3.
</verify>
