- `ulid`: Text unique, not null, default=lambda: str(ULID())

- `webhook_url`: Text nullable
- `webhook_secret`: Text nullable
- `total_items`: Integer not null, server_default="0"

- `concurrency_limit`: Integer not null, server_default="10"

- `__tablename__ = "batch_items"`

- `ulid`: Text unique, not null, default=lambda: str(ULID())

- `filename`: Text not null

- `rule_pack`: Text nullable
- `priority`: Text not null, server_default="normal"

- `error_message`: Text nullable
- `file_size_bytes`: BigInteger nullable
- `duration_ms`: Float nullable

- `started_at`: TIMESTAMP(timezone=True) nullable
- `completed_at`: TIMESTAMP(timezone=True) nullable
- Relationships: `batch` -> Batch (back_populates="items"), `analysis` -> Analysis
- Indexes: `ix_batch_items_batch_status` on (batch_id, status), `ix_batch_items_batch_created` on (batch_id, created_at)

**WebhookDelivery model:**

- Relationships: `batch` -> Batch (back_populates="webhook_deliveries")
- Index: `ix_webhook_deliveries_retry` on (status, next_retry_at)

Create `backend/src/services/batch_service.py` with these functions:

**Constants (from env vars):**
```python
BATCH_MAX_ITEMS = int(os.getenv("BATCH_MAX_ITEMS", "10000"))
BATCH_MAX_ZIP_BYTES = int(os.getenv("BATCH_MAX_ZIP_BYTES", str(5 * 1024**3)))
BATCH_MAX_FILE_BYTES = int(os.getenv("BATCH_MAX_FILE_BYTES", str(100 * 1024**2)))
MAX_COMPRESSION_RATIO = 100
DEFAULT_BATCH_CONCURRENCY = int(os.getenv("DEFAULT_BATCH_CONCURRENCY", "10"))
BATCH_BLOB_DIR = os.getenv("BATCH_BLOB_DIR", "/data/blobs/batch")
VALID_EXTENSIONS = {".stl", ".step", ".stp"}
```

**`create_batch(session, user, input_mode, webhook_url, webhook_secret, concurrency_limit) -> Batch`:**
- Creates a Batch row with status="pending", returns the Batch object
- Uses ULID for public ID
- Sets concurrency_limit from arg or DEFAULT_BATCH_CONCURRENCY

**`extract_zip_to_items(zip_bytes, batch_ulid) -> list[dict]`:**
- Extracts valid CAD files from ZIP archive to `/data/blobs/batch/{batch_ulid}/`
- Enforces: max items (BATCH_MAX_ITEMS), per-file size (BATCH_MAX_FILE_BYTES), compression ratio (MAX_COMPRESSION_RATIO)
- Skips non-CAD files silently, skips oversized files with status="skipped"
- Returns list of dicts: `{"filename": str, "path": str, "size": int}` or `{"filename": str, "status": "skipped", "error": str}`
- Uses `zipfile.ZipFile` with `ZipInfo.file_size` pre-check (D-05)
- Path traversal prevention: `os.path.basename()` on all filenames

**`parse_csv_manifest(csv_content: str) -> list[dict]`:**
- Parses CSV with columns: filename (required), process_types (optional), rule_pack (optional), priority (optional, default "normal")
- Returns list of dicts with parsed values
- Raises ValueError on missing filename column or invalid priority values

**`create_batch_items(session, batch_id, items_data) -> int`:**
- Bulk-inserts BatchItem rows from parsed manifest/extraction data
- Returns count of items created
- Sets status="pending" for valid items, status="skipped" for invalid

**`get_batch_progress(session, batch_ulid, user_id) -> dict | None`:**
- Returns batch progress dict matching D-18 response format
- Single-row SELECT from batches table (O(1) via denormalized counters)
- Returns None if batch not found or not owned by user

**`get_batch_items_page(session, batch_id, status_filter, cursor, limit) -> tuple[list[BatchItem], bool]`:**
- Cursor-paginated batch items query (D-19 pattern)
- Optional status filter
- Returns (items, has_more)

**`generate_results_csv(session, batch_id) -> AsyncGenerator[str, None]`:**
- Async generator yielding CSV rows for StreamingResponse
- Header row: filename,status,verdict,best_process,issue_count,duration_ms,analysis_url,error
- Paginated internal query (200 items per page) to avoid memory bloat
- Joins batch_items with analyses for verdict/best_process data

**`update_batch_counters(session, batch_id, field) -> None`:**
- Atomic SQL increment: `UPDATE batches SET {field} = {field} + 1 WHERE id = :batch_id`
- field is "completed_items" or "failed_items"

**`cleanup_batch_files(batch_ulid: str) -> None`:**
- Deletes `/data/blobs/batch/{batch_ulid}/` directory
- Called by cleanup task after retention period
</action>

## Security Threat Model

| ID | Threat | Severity | Mitigation | Task |
|----|--------|----------|------------|------|
| T-09-01 | ZIP bomb exhausts server memory/disk | HIGH | Compression ratio >100:1 rejection, per-file size limit, total file count cap, Content-Length pre-check | 09-A-03 |
| T-09-02 | Path traversal via ZIP entry names | HIGH | `os.path.basename()` strips directory components from all filenames before extraction | 09-A-03 |
| T-09-03 | Webhook secret exposed in API responses | MEDIUM | Never include webhook_secret in GET responses; only accept on POST | 09-B-02 |
| T-09-04 | Unauthorized batch access | MEDIUM | All batch endpoints filter by user_id from authenticated session | 09-B-02 |
| T-09-05 | Webhook replay attack | MEDIUM | Timestamp in HMAC signature + 5-minute tolerance window | 09-B-03 |
</threat_model>
