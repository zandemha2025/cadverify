<action>
Create `backend/src/services/webhook_service.py` with:

**`sign_webhook_payload(payload_bytes: bytes, secret: str) -> str`:**
- Computes HMAC-SHA256 signature using Stripe-like format
- Returns header value: `t={unix_timestamp},v1={hex_signature}`
- `signed_content = f"{timestamp}.{payload_bytes.decode()}"`
- Uses `hmac.new(secret.encode(), signed_content.encode(), hashlib.sha256).hexdigest()`

**`verify_webhook_signature(payload_bytes: bytes, secret: str, signature_header: str, tolerance_sec: int = 300) -> bool`:**
- Parses `t=...` and `v1=...` from header
- Reconstructs signed_content and computes expected signature
- Uses `hmac.compare_digest()` for timing-safe comparison
- Rejects if `abs(time.time() - int(t)) > tolerance_sec` (replay protection)

**`async def create_webhook_delivery(session, batch_id, event_type, payload) -> WebhookDelivery`:**
- Creates a WebhookDelivery row with status="pending", attempts=0
- Sets payload_json from the payload dict

**`async def deliver_webhook(session, delivery_id) -> bool`:**
- Fetches delivery + associated batch (for webhook_url, webhook_secret)
- If batch has no webhook_url, mark delivered and return True
- Signs payload with `sign_webhook_payload()`
- POSTs to webhook_url using `httpx.AsyncClient` with 10-second timeout
- Headers: `Content-Type: application/json`, `X-CadVerify-Signature: {signature}`, `User-Agent: CadVerify-Webhook/1.0`
- On 2xx: update delivery status="delivered", response_code, last_attempt_at; return True
- On non-2xx or exception: increment attempts, set response_code (if available), set last_attempt_at; return False

**`async def schedule_webhook_retry(session, delivery_id, pool) -> None`:**
- RETRY_DELAYS = [10, 30, 90, 270, 810]
- Reads current attempts count from delivery
- If attempts >= 5: mark status="failed", return
- Calculates delay = RETRY_DELAYS[attempts] + random.uniform(0, RETRY_DELAYS[attempts] * 0.1)
- Sets next_retry_at = now() + timedelta(seconds=delay)
- Enqueues `dispatch_webhook` arq task with `_defer_by=delay`

**Imports:** hashlib, hmac, time, random, logging, os, httpx, datetime, sqlalchemy, ulid
</action>

```python
router = APIRouter(prefix="/api/v1", tags=["batch"])
```

**`POST /batch` (status_code=202):**
- Accepts multipart/form-data (ZIP file upload) OR JSON body (S3 reference)
- For ZIP mode: read file from `UploadFile`, validate Content-Length <= BATCH_MAX_ZIP_BYTES
- For S3 mode: accept JSON `{"s3_bucket": str, "s3_prefix": str, "manifest_url": str | None}`
- Optional form/body fields: `webhook_url`, `webhook_secret`, `manifest` (CSV file), `concurrency_limit` (int)
- Calls `batch_service.create_batch()` to create Batch row
- For ZIP: calls `batch_service.extract_zip_to_items()` to extract, then `create_batch_items()`
- For S3: if manifest provided, parses CSV; else creates items from S3 listing (via boto3)
- Enqueues `run_batch_coordinator` arq task with batch ULID
- Returns 202 with `{"batch_id": batch.ulid, "status": "pending", "status_url": f"/api/v1/batch/{batch.ulid}"}`
- Auth: `Depends(require_api_key)`
- **Never include webhook_secret in any response** (T-09-03)

**`GET /batch/{batch_id}` (status_code=200):**
- Calls `batch_service.get_batch_progress(session, batch_id, user.user_id)`
- Returns D-18 progress response format
- 404 if batch not found or not owned by user (D-12 pattern: 404 not 403)
- Auth: `Depends(require_api_key)`

**`GET /batch/{batch_id}/items` (status_code=200):**
- Query params: `status: Optional[str]`, `cursor: Optional[str]`, `limit: int = Query(default=50, le=200)`
- Calls `batch_service.get_batch_items_page()`
- Returns D-19 paginated response format
- 404 if batch not found or not owned by user
- Auth: `Depends(require_api_key)`

**`GET /batch/{batch_id}/results/csv` (status_code=200):**
- Returns StreamingResponse with CSV content
- Content-Disposition header: `attachment; filename=batch_{batch_id}_results.csv`
- media_type: "text/csv"
- 404 if batch not found; 400 if batch not yet completed
- Auth: `Depends(require_api_key)`

**`POST /batch/{batch_id}/cancel` (status_code=200):**
- Sets batch status to "cancelled"
- Cancels all pending/queued items (set status="skipped")
- Does not cancel in-progress items (they complete naturally)
- Returns `{"batch_id": batch.ulid, "status": "cancelled"}`
- 404 if batch not found or not owned by user
- 409 if batch already in terminal state (completed, failed, cancelled)
- Auth: `Depends(require_api_key)`
</action>

<action>
Create `backend/src/jobs/batch_tasks.py` with three arq task functions:

**`async def run_batch_coordinator(ctx: dict, batch_ulid: str) -> None`:**
1. Get session from `get_session_factory()()`
2. Load Batch row by ULID
3. Set status="extracting", started_at=now(); commit
4. If input_mode="zip": files already extracted by the route handler, items already created
5. If input_mode="s3": fetch manifest from S3 (boto3), parse CSV, create batch items, then fetch each file on demand during item processing
6. Set status="processing", total_items=count of items; commit
7. **Coordinator loop:**
   - Query batch_items with status="pending", ordered by priority DESC (high before normal), created_at ASC
   - Count currently active items (status in ["queued", "processing"])
   - If active < concurrency_limit and pending items exist: enqueue next batch as `run_batch_item` tasks, set their status="queued"
   - High-priority items use `_defer_by=0`, normal items use `_defer_by=1`
   - Sleep 2 seconds (`await asyncio.sleep(2)`)
   - Check if `completed_items + failed_items == total_items`: if yes, break loop
   - Check if batch.status == "cancelled": if yes, break loop
8. On loop exit (all done): compute summary stats, set status="completed", completed_at=now()
9. If batch has webhook_url: create batch.completed webhook delivery and enqueue dispatch
10. Handle exceptions: set status="failed", log error

**`async def run_batch_item(ctx: dict, item_ulid: str) -> None`:**
1. Get session
2. Load BatchItem by ULID, load associated Batch
3. Set item status="processing", started_at=now(); commit
4. Read file bytes: if input_mode="zip", read from `/data/blobs/batch/{batch_ulid}/{filename}`; if "s3", fetch from S3
5. Construct AuthedUser from batch.user_id and batch.api_key_id
6. Call `analysis_service.run_analysis(file_bytes, filename, process_types, rule_pack, user, session)`
7. On success: set item status="completed", analysis_id from returned analysis, duration_ms, completed_at; call `update_batch_counters(session, batch_id, "completed_items")`
8. On failure: set item status="failed", error_message=str(exception); call `update_batch_counters(session, batch_id, "failed_items")`
9. If batch has webhook_url: create batch_item.completed webhook delivery with D-10 payload format; enqueue `dispatch_webhook`

**`async def dispatch_webhook(ctx: dict, delivery_id: int) -> None`:**
1. Get session
2. Call `webhook_service.deliver_webhook(session, delivery_id)`
3. If delivery failed: call `webhook_service.schedule_webhook_retry(session, delivery_id, ctx["redis"])`

Use `arq.cron` or `@func` decorator on `run_batch_coordinator` with `timeout=14400` (4 hours).
</action>
