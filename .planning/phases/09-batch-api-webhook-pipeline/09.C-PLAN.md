<action>
Create `frontend/lib/api/batch.ts` with TypeScript API client functions:

**Types:**
```typescript
interface BatchCreateResponse {
  batch_id: string;
  status: string;
  status_url: string;
}

interface BatchProgress {
  batch_id: string;
  status: "pending" | "extracting" | "processing" | "completed" | "failed" | "cancelled";
  input_mode: "zip" | "s3";
  total_items: number;
  completed_items: number;
  failed_items: number;
  in_progress_items: number;
  pending_items: number;
  created_at: string;
  started_at: string | null;
  completed_at: string | null;
  summary: BatchSummary | null;
}

interface BatchSummary {
  total: number;
  completed: number;
  failed: number;
  pass: number;
  issues: number;
  fail: number;
  duration_sec: number;
}

interface BatchItem {
  item_id: string;
  filename: string;
  status: string;
  verdict: string | null;
  analysis_id: string | null;
  duration_ms: number | null;
  error_message: string | null;
}

interface BatchItemsResponse {
  items: BatchItem[];
  next_cursor: string | null;
  has_more: boolean;
}
```

**Functions:**
- `createBatch(file: File, options?: { webhookUrl?: string, manifest?: File, concurrencyLimit?: number }): Promise<BatchCreateResponse>` — POST /api/v1/batch with FormData
- `createBatchS3(params: { s3Bucket: string, s3Prefix: string, manifestUrl?: string, webhookUrl?: string, webhookSecret?: string, concurrencyLimit?: number }): Promise<BatchCreateResponse>` — POST /api/v1/batch with JSON body
- `getBatchProgress(batchId: string): Promise<BatchProgress>` — GET /api/v1/batch/{id}
- `getBatchItems(batchId: string, options?: { status?: string, cursor?: string, limit?: number }): Promise<BatchItemsResponse>` — GET /api/v1/batch/{id}/items
- `downloadBatchCsv(batchId: string): Promise<Blob>` — GET /api/v1/batch/{id}/results/csv
- `cancelBatch(batchId: string): Promise<{ batch_id: string, status: string }>` — POST /api/v1/batch/{id}/cancel

All functions include Authorization header from session/API key. Handle 404, 409, 4xx with typed errors.
</action>
