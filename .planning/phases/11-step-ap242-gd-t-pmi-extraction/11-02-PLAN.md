<objective>
Build the GD&T/PMI extraction module that traverses an AP242Document's XDE DimTolTool to extract geometric tolerances, datums, dimensional tolerances, and surface finish annotations. Produces structured ToleranceEntry objects using a Pydantic/dataclass model mirroring ISO 1101 tolerance types. This is the bridge between OCP's raw PMI data and CadVerify's analysis pipeline.
</objective>

<must_haves>
- Tolerance data model (ToleranceEntry, ToleranceReport) with ISO 1101 types
- GD&T extractor that reads DimTolTool labels and produces ToleranceEntry list
- Datum reference extraction (labels A, B, C, etc.)
- Surface finish (Ra) extraction when present in PMI
- Partial extraction support (skip unreadable annotations with warning, extract remainder)
</must_haves>

<title>Implement GD&T extractor using OCP XDE DimTolTool</title>
<files>backend/src/parsers/gdt_extractor.py</files>
<read_first>
- backend/src/parsers/step_ap242_parser.py (AP242Document structure тАФ this is the input)
- backend/src/analysis/tolerance_models.py (ToleranceEntry тАФ this is the output)
- backend/src/analysis/models.py (Severity enum for partial-extraction warnings)
</read_first>
<action>
Create `backend/src/parsers/gdt_extractor.py` with:

1. Feature gating:
   ```python
   _HAS_XDE = False
   try:
       from OCP.XCAFDimTolObjects import (
           XCAFDimTolObjects_GeomToleranceObject,
           XCAFDimTolObjects_DimensionObject,
           XCAFDimTolObjects_DatumObject,
       )
       from OCP.XCAFDoc import XCAFDoc_DimTolTool
       from OCP.TDF import TDF_LabelSequence
       _HAS_XDE = True
   except ImportError:
       pass
   ```

2. `_OCP_TYPE_MAP: dict` mapping OCP `GeomToleranceType` enum values to `ToleranceType` enum values. Cover all 14 types (flatness, straightness, circularity, cylindricity, parallelism, perpendicularity, angularity, position, concentricity, symmetry, profile_of_surface, profile_of_line, circular_runout, total_runout).

3. `extract_gdt(ap242_doc: AP242Document) -> tuple[list[ToleranceEntry], list[str]]` function:
   - Returns `(tolerances, warnings)`.
   - Get GDT labels from `ap242_doc.dim_tol_tool.GetGDTLabels()`.
   - For each label, try to extract tolerance object:
     a. Attempt `GetGeomTolerance(label)` тАФ if successful, extract type, value, datum refs.
     b. Attempt `GetDimension(label)` тАФ if successful, extract upper/lower deviations.
     c. Attempt `GetDatum(label)` тАФ if successful, store datum label mapping.
   - Auto-generate `tolerance_id` as "TOL-001", "TOL-002", etc.
   - Map OCP tolerance type to `ToleranceType` via `_OCP_TYPE_MAP`.
   - Extract datum references by following datum label links.
   - On extraction failure for any single annotation: append warning string, continue to next.
   - Return list of successfully extracted `ToleranceEntry` objects + list of warning strings.

4. `extract_surface_finish(ap242_doc: AP242Document) -> list[tuple[str, float]]` function:
   - Attempt to read surface finish annotations from PMI.
   - Return list of (feature_description, ra_um) tuples.
   - If no surface finish found, return empty list.

Logging via `logging.getLogger("cadverify.gdt_extractor")`.
</action>
