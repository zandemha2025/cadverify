Build the tolerance validation engine: YAML-based process capability tables covering all 21 manufacturing processes, a loader that maps capabilities to ProcessType, and a tolerance_service that orchestrates AP242 parse -> GD&T extract -> validate against capabilities -> produce ToleranceReport. Also integrates with analysis_service.py to run tolerance analysis alongside existing mesh analysis for STEP files.
</objective>

<must_haves>
- YAML capability tables with achievable_min for each tolerance type per process
- Surface finish Ra limits per process
- Tolerance validation logic: achievable / marginal / not_achievable verdicts
- tolerance_service orchestration: parse -> extract -> validate -> ToleranceReport
- Integration into analysis_service.py for automatic AP242 detection on STEP uploads
- Summary score calculation (% of tolerances achievable by best process)
</must_haves>

<action>
Create `backend/src/analysis/capabilities/loader.py` with:

1. `ProcessCapability` dataclass:
   - `achievable_min: dict[str, float]` — maps tolerance_type.value to minimum achievable tolerance in mm
   - `ra_min_um: float` — minimum achievable surface finish Ra
   - `ra_typical_um: float` — typical surface finish Ra

2. `_CAPABILITIES: dict[str, ProcessCapability] | None = None` — module-level cache (loaded once).

3. `load_capabilities() -> dict[str, ProcessCapability]` function:
   - Read `process_tolerances.yaml` from same directory using `Path(__file__).parent / "process_tolerances.yaml"`.
   - Parse YAML via `yaml.safe_load`.
   - For each process key, flatten the nested category structure into a flat `achievable_min` dict:
     ```python
     # e.g., {"flatness": 0.005, "straightness": 0.005, "position": 0.01, ...}
     ```
   - Validate all 21 ProcessType values have entries; log warning for any missing.
   - Cache in `_CAPABILITIES` and return.

4. `get_capability(process_type: ProcessType) -> ProcessCapability | None` function:
   - Call `load_capabilities()` if not cached.
   - Return capability for `process_type.value`, or None if missing.

5. `validate_tolerance(tolerance_value_mm: float, process_type: ProcessType, tolerance_type_str: str) -> tuple[str, float, float]` function:
   - Returns `(verdict, capability_min, margin)`.
   - Look up achievable_min for the given process + tolerance type.
   - If tolerance_value_mm >= achievable_min * 2: verdict = "achievable"
   - If tolerance_value_mm >= achievable_min: verdict = "marginal"
   - If tolerance_value_mm < achievable_min: verdict = "not_achievable"
   - margin = tolerance_value_mm - achievable_min
   - If no capability data for this combination: verdict = "unknown", capability_min = 0, margin = 0.

Import `yaml` (add to requirements if needed — already available via `pyyaml`).
</action>

   - Serialize to JSON-compatible dict matching the response format from RESEARCH.md section 5.
