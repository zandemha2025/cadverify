# Plan 12.A: SSO/SAML Integration

<objective>
Implement SAML 2.0 Service Provider authentication using python3-saml so enterprise customers can authenticate via their corporate Identity Provider (Okta, Azure AD, PingFederate) instead of Google OAuth. Controlled by `AUTH_MODE` env var (saml/google/hybrid).
</objective>

<must_haves>
- Enterprise customer can configure their SAML IdP and users authenticate via SSO without Google OAuth
- `AUTH_MODE` env var switches between saml, google, and hybrid modes
- SP metadata endpoint enables one-click IdP configuration
- SAML-authenticated users are auto-provisioned with a `users` row and API key
</must_haves>

<task id="12.A.2" type="execute">
<title>Implement SAML SP module</title>
<read_first>
- backend/src/auth/oauth.py
- backend/src/auth/models.py
- backend/src/auth/hashing.py
- backend/src/auth/dashboard_session.py
</read_first>
<action>
1. Create `backend/src/auth/saml.py` with:
   - `_build_request_data(request: Request) -> dict` that extracts https, http_host, server_port, script_name, get_data, post_data from Starlette Request.
   - `_load_saml_settings() -> dict` that reads `saml/settings.json` and `saml/advanced_settings.json` from path `os.getenv("SAML_CONFIG_DIR", "saml/")`, also loads `saml/idp_metadata.xml` if present.
   - `router = APIRouter(prefix="/auth/saml")` with 5 endpoints:
     a. `GET /login` -- build `OneLogin_Saml2_Auth`, call `auth.login()`, redirect to IdP
     b. `POST /acs` -- build auth, call `auth.process_response()`, validate, extract NameID (email), call `_saml_provision_user(email)`, set session cookie via `set_session_cookie()`, redirect to dashboard
     c. `GET /logout` -- build auth, call `auth.logout()`, redirect to IdP SLO
     d. `POST /sls` -- build auth, call `auth.process_slo()`, clear session, redirect to login
     e. `GET /metadata` -- build auth, return `auth.get_settings().get_sp_metadata()` as XML response with content-type `application/xml`
   - `_saml_provision_user(email: str) -> User` that calls existing `upsert_user()` from `auth/models.py` with `auth_provider='saml'`, then calls `mint_token()` if user has no API keys. Set default role to `viewer`.

2. Add `auth_provider` column to User model in `backend/src/auth/models.py`:
   - `auth_provider: Mapped[str] = mapped_column(Text, server_default="google", nullable=False)`
   - This column distinguishes SAML vs Google OAuth users.

3. Add AUTH_MODE gating in `backend/src/api/app.py`:
   - Read `AUTH_MODE = os.getenv("AUTH_MODE", "google")`
   - If AUTH_MODE in ("saml", "hybrid"): include `saml.router`
   - If AUTH_MODE in ("google", "hybrid"): include `oauth.router`
   - If AUTH_MODE == "saml": do NOT include `oauth.router` or `magic_link.router`
</action>
