## Implementation Details

### SAML SP Module (saml.py)
- 5 endpoints: `/auth/saml/{login,acs,logout,sls,metadata}`
- `_build_request_data` extracts Starlette request info for python3-saml
- `_saml_provision_user` upserts user via existing `upsert_user()`, mints API key
- Error handling with structured error responses matching project conventions

### AUTH_MODE Gating (main.py)
- `AUTH_MODE=google` (default): only Google OAuth + magic link routes
- `AUTH_MODE=saml`: only SAML routes
- `AUTH_MODE=hybrid`: both SAML and Google OAuth routes

### Database Changes
- `auth_provider` column (TEXT, default 'google') on users table
- `role` column (TEXT, default 'analyst') with CHECK constraint (viewer/analyst/admin)
- ORM model updated in db/models.py
