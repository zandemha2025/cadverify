# Plan 12.B: RBAC Middleware + Permission Matrix

<objective>
Implement role-based access control with three roles (viewer, analyst, admin) enforced via FastAPI dependency injection. Viewers can only read; analysts can trigger analyses; admins can manage users and view audit logs. Add admin API endpoints for user/role management.
</objective>

<must_haves>
- Three roles (viewer, analyst, admin) enforce distinct permissions
- Viewer cannot trigger analysis, analyst cannot manage users
- Admin API for listing users and assigning roles
- RBAC composes with existing require_api_key dependency
</must_haves>

<tasks>

<task id="12.B.1" type="execute">
<title>Create RBAC module with Role enum and require_role dependency</title>
<read_first>
- backend/src/auth/require_api_key.py
- backend/src/db/models.py
</read_first>
<action>
1. Create `backend/src/auth/rbac.py` with:
   - `class Role(str, Enum)` with values `viewer = "viewer"`, `analyst = "analyst"`, `admin = "admin"` and a `rank` property: viewer=1, analyst=2, admin=3.
   - `def require_role(min_role: Role)` that returns an async dependency:
     ```python
     def require_role(min_role: Role):
         async def _check(user: AuthedUser = Depends(require_api_key)) -> AuthedUser:
             if Role(user.role).rank < min_role.rank:
                 raise HTTPException(
                     403,
                     detail={
                         "code": "insufficient_role",
                         "message": f"Requires {min_role.value} role or higher",
                         "doc_url": f"https://docs.cadverify.com/errors#insufficient_role",
                     },
                 )
             return user
         return _check
     ```

2. Update `AuthedUser` in `backend/src/auth/require_api_key.py`:
   - Add `role: str` field to `AuthedUser` model.
   - In `require_api_key()`, after validating the key, query user's role from the DB and set `user.role`.
</action>
<acceptance_criteria>
- `backend/src/auth/rbac.py` exists and contains `class Role`
- `backend/src/auth/rbac.py` contains `def require_role`
- `backend/src/auth/require_api_key.py` AuthedUser contains `role: str`
- grep for `Role.viewer` in rbac.py returns a match
- grep for `insufficient_role` in rbac.py returns a match
</acceptance_criteria>
</task>

<action>

   - READ endpoints (GET /analyses, GET /analyses/{id}, GET /batch/{id}, GET /processes, etc.): `Depends(require_role(Role.viewer))` -- all roles can read
   - WRITE endpoints (POST /validate, POST /validate/repair, POST /batch, POST /reconstruct): `Depends(require_role(Role.analyst))` -- analyst and admin only
   - API key management (POST /keys, DELETE /keys/{id}, etc.): `Depends(require_role(Role.analyst))` -- analyst and admin only

</action>
