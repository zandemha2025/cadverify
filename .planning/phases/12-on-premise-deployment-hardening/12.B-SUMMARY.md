## Implementation Details

### Permission Matrix

| Endpoint Pattern | Min Role | Examples |
|-----------------|----------|----------|
| GET (read-only) | viewer | /processes, /materials, /analyses, /batch/{id} |
| POST (write) | analyst | /validate, /validate/repair, /batch, /reconstruct |
| Admin API | admin | /admin/users, /admin/users/{id}/role |

### Key Design Decisions

1. **Hierarchical rank model**: viewer=1, analyst=2, admin=3. Higher rank includes all lower permissions.
2. **Dependency composition**: `require_role` wraps `require_api_key` via `Depends()`, so auth validation happens first, then role check.
3. **lookup_user_role**: New DB query in auth/models.py fetches user role column. Default fallback is "analyst" for backward compatibility.
4. **Self-demotion guard**: Admin cannot PATCH their own role (returns 400).
