# Plan 12.C: Audit Logging Service + Schema

<objective>
Implement a comprehensive audit logging service that records every analysis action with user identity, file hash, timestamp, and result summary. Audit log entries are stored in a Postgres `audit_log` table with an admin export endpoint for compliance workflows.
</objective>

<must_haves>
- Every analysis action produces an audit log entry with user identity, file hash, timestamp, and result summary
- Audit log is append-only (no UPDATE/DELETE operations)
- Admin can export audit logs via API with time-range filtering
- Audit entries cover authentication, analysis, batch, sharing, key management, and role changes
</must_haves>

   - `async def export_audit_csv(start: datetime, end: datetime, user_id: int | None = None, action: str | None = None) -> str`:
     - Returns CSV string of audit entries matching filters. Columns: timestamp, user_email, action, resource_type, resource_id, ip_address, file_hash, result_summary.
