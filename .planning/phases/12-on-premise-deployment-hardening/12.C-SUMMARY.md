## Decisions Made

1. Denormalized user_email stored directly on audit_log row so entries survive user deletion (FK uses SET NULL)
2. _lookup_email helper resolves email from DB since AuthedUser lacks email field
3. All audit writes use fire-and-forget pattern with error swallowing to prevent audit failures from breaking API requests
