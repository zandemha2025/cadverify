# Plan 12.D: Air-Gapped Docker Compose + Helm Chart

<objective>
Create a self-contained air-gapped Docker Compose deployment bundle and a Helm chart for Kubernetes deployment. Both must work with zero external network calls at runtime, support SAML/RBAC/audit configuration, and enable enterprise IT to deploy CadVerify in isolated networks.
</objective>

<must_haves>
- `docker compose up` in an air-gapped network (no internet) produces a fully functional system
- Helm chart deploys to Kubernetes with configurable replicas, resource limits, and persistent volumes
- All Docker images pre-bundled as tar files for air-gap loading
- SAML config mountable as volume/ConfigMap
</must_haves>

<tasks>

<task id="12.D.1" type="execute">
<title>Create air-gapped Docker Compose bundle</title>
<read_first>
- docker-compose.yml
- backend/Dockerfile
</read_first>
<action>
1. Create `cadverify-enterprise/` directory with:

2. `cadverify-enterprise/docker-compose.yml`:
   - Copy from root docker-compose.yml but with these changes:
     - All services use `image:` instead of `build:` (pre-built images)
     - Backend image: `cadverify-backend:latest`
     - Worker image: `cadverify-backend:latest` (same image, different command)
     - Frontend image: `cadverify-frontend:latest`
     - Postgres: `postgres:16-alpine`
     - Redis: `redis:7-alpine`
   - Add SAML volume mount on backend and worker: `- ./saml:/app/saml:ro`
   - Add environment variables: `AUTH_MODE`, `SAML_ENABLED`, `SAML_CONFIG_DIR=/app/saml`
   - Named volumes: `pgdata`, `redis-data`, `blobs`
   - Backend volume mount: `- blobs:/data/blobs`

3. `cadverify-enterprise/.env.example`:
   - Document ALL env vars with comments:
     ```
     # === Database ===
     POSTGRES_PASSWORD=changeme
     DATABASE_URL=postgresql+asyncpg://cadverify:${POSTGRES_PASSWORD}@postgres:5432/cadverify
     
     # === Auth Mode ===
     # Options: google, saml, hybrid
     AUTH_MODE=saml
     
     # === SAML (when AUTH_MODE=saml or hybrid) ===
     SAML_ENABLED=true
     SAML_SP_ENTITY_ID=https://cadverify.example.com
     SAML_SP_ACS_URL=https://cadverify.example.com/auth/saml/acs
     SAML_SP_SLO_URL=https://cadverify.example.com/auth/saml/sls
     SAML_CONFIG_DIR=/app/saml
     
     # === Redis ===
     REDIS_URL=redis://redis:6379/0
     
     # === API ===
     DASHBOARD_ORIGIN=https://cadverify.example.com
     API_KEY_PEPPER=generate-a-random-32-byte-hex-string
     
     # === Batch Processing ===
     DEFAULT_BATCH_CONCURRENCY=10
     BATCH_MAX_ITEMS=10000
     ```

4. `cadverify-enterprise/scripts/load-images.sh`:
   ```bash
   #!/bin/bash
   set -e
   echo "Loading CadVerify Docker images..."
   for tar in images/*.tar; do
     echo "Loading $tar..."
     docker load < "$tar"
   done
   echo "All images loaded successfully."
   docker images | grep -E "cadverify|postgres|redis"
   ```

5. `cadverify-enterprise/scripts/export-images.sh` (run on build machine, not air-gapped):
   ```bash
   #!/bin/bash
   set -e
   mkdir -p images
   docker save cadverify-backend:latest > images/cadverify-backend.tar
   docker save cadverify-frontend:latest > images/cadverify-frontend.tar
   docker save postgres:16-alpine > images/postgres-16-alpine.tar
   docker save redis:7-alpine > images/redis-7-alpine.tar
   echo "Images exported to images/ directory"
   ```

6. `cadverify-enterprise/scripts/init-db.sh`:
   ```bash
   #!/bin/bash
   set -e
   echo "Running database migrations..."
   docker compose exec backend alembic upgrade head
   echo "Migrations complete."
   ```

7. `cadverify-enterprise/scripts/health-check.sh`:
   ```bash
   #!/bin/bash
   echo "Checking CadVerify services..."
   curl -sf http://localhost:8000/health && echo "Backend: OK" || echo "Backend: FAIL"
   curl -sf http://localhost:3000 > /dev/null && echo "Frontend: OK" || echo "Frontend: FAIL"
   docker compose exec postgres pg_isready -U cadverify && echo "Postgres: OK" || echo "Postgres: FAIL"
   docker compose exec redis redis-cli ping | grep -q PONG && echo "Redis: OK" || echo "Redis: FAIL"
   ```

8. Copy `saml/settings.json.template` and `saml/advanced_settings.json.template` into `cadverify-enterprise/saml/`.
</action>

<task id="12.D.2" type="execute">
<title>Create Helm chart structure and templates</title>
<read_first>
- docker-compose.yml
- cadverify-enterprise/docker-compose.yml
</read_first>
<action>
1. Create `charts/cadverify/Chart.yaml`:
   ```yaml
   apiVersion: v2
   name: cadverify
   description: CadVerify DFM Analysis Platform
   type: application
   version: 0.1.0
   appVersion: "2.0.0"
   ```

2. Create `charts/cadverify/values.yaml`:
   ```yaml
   replicaCount:
     backend: 2
     worker: 2
     frontend: 2

   image:
     backend:
       repository: cadverify-backend
       tag: latest
       pullPolicy: IfNotPresent
     frontend:
       repository: cadverify-frontend
       tag: latest
       pullPolicy: IfNotPresent

   resources:
     backend:
       requests: { cpu: "250m", memory: "512Mi" }
       limits: { cpu: "1000m", memory: "2Gi" }
     worker:
       requests: { cpu: "500m", memory: "1Gi" }
       limits: { cpu: "2000m", memory: "4Gi" }
     frontend:
       requests: { cpu: "100m", memory: "128Mi" }
       limits: { cpu: "500m", memory: "512Mi" }

   persistence:
     blobs:
       enabled: true
       size: 50Gi
       storageClass: ""
     postgres:
       enabled: true
       size: 20Gi
       storageClass: ""

   saml:
     enabled: false
     idpMetadata: ""
     spEntityId: ""
     spAcsUrl: ""
     spSloUrl: ""

   auth:
     mode: "google"

   audit:
     enabled: true
     retentionDays: 365

   rbac:
     defaultRole: "analyst"

   ingress:
     enabled: false
     className: "nginx"
     host: "cadverify.example.com"
     tls: []

   postgresql:
     host: "postgres"
     port: 5432
     database: "cadverify"
     username: "cadverify"

   redis:
     host: "redis"
     port: 6379
   ```

3. Create `charts/cadverify/values-enterprise.yaml`:
   ```yaml
   # Enterprise overlay: SAML, air-gapped, audit
   image:
     backend:
       pullPolicy: Never
     frontend:
       pullPolicy: Never
   auth:
     mode: "saml"
   saml:
     enabled: true
   audit:
     enabled: true
   rbac:
     defaultRole: "viewer"
   ```

4. Create `charts/cadverify/templates/_helpers.tpl` with standard name/fullname/labels helpers.

5. Create deployment templates:
   - `templates/deployment-backend.yaml`: Deployment with backend container, env from secrets/configmaps, SAML volume mount (conditional on saml.enabled), blobs PVC mount, liveness/readiness probes on /health.
   - `templates/deployment-worker.yaml`: Deployment with same image as backend, command `["arq", "src.jobs.worker.WorkerSettings"]`, same env/volumes.
   - `templates/deployment-frontend.yaml`: Deployment for frontend with NEXT_PUBLIC_API_URL env var.

6. Create service templates:
   - `templates/service-backend.yaml`: ClusterIP service on port 8000.
   - `templates/service-frontend.yaml`: ClusterIP service on port 3000.

7. Create `templates/ingress.yaml` (conditional on ingress.enabled): Ingress with backend and frontend paths.

8. Create config/secret templates:
   - `templates/configmap-saml.yaml` (conditional on saml.enabled): ConfigMap with SAML settings from values.
   - `templates/secret-db.yaml`: Secret with DATABASE_URL, POSTGRES_PASSWORD.
   - `templates/secret-saml.yaml` (conditional on saml.enabled): Secret with SP key.

9. Create `templates/pvc-blobs.yaml`: PVC for blob storage with configurable size.

10. Create `templates/job-migrate.yaml`: Kubernetes Job that runs `alembic upgrade head` as a pre-install/pre-upgrade hook.
</action>
