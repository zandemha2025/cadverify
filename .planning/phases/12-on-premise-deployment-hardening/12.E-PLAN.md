</objective>

<action>
Create `cadverify-enterprise/docs/ENTERPRISE-SETUP.md` with these sections:

**1. Prerequisites**
- Docker 24+ and Docker Compose v2 (for Docker deployment)
- Kubernetes 1.28+ and Helm 3.12+ (for Kubernetes deployment)
- Minimum hardware: 4 CPU cores, 8 GB RAM, 100 GB disk
- Network: internal DNS, TLS certificate for the CadVerify domain
- SAML IdP with SAML 2.0 SP support (Okta, Azure AD, PingFederate, or compatible)

**2. Quick Start (Docker Compose)**
- Step-by-step: copy bundle, load images, copy .env.example to .env, edit .env, docker compose up, run init-db.sh, verify with health-check.sh
- Expected time: 5-10 minutes

**3. SSO/SAML Configuration**
- 3a. Generate SP certificate: `openssl req -new -x509 -days 3650 -nodes -out saml/sp.crt -keyout saml/sp.key -subj "/CN=cadverify-sp"`
- 3b. Configure saml/settings.json from template (fill SP entity ID, ACS URL, SLO URL)
- 3c. Okta setup: Create SAML 2.0 app > Single sign-on URL = ACS URL > Audience URI = SP Entity ID > Attribute statements: email=user.email > Download IdP metadata XML > Save as saml/idp_metadata.xml
- 3d. Azure AD (Entra ID) setup: Enterprise Applications > New > SAML SSO > Basic SAML Configuration: Identifier = SP Entity ID, Reply URL = ACS URL > Download Federation Metadata XML > Save as saml/idp_metadata.xml
- 3e. PingFederate setup: SP Connections > Create > Import SP metadata from /auth/saml/metadata > Configure attribute contract (email)
- 3f. Set AUTH_MODE=saml in .env, restart services
- 3g. Verify: visit /auth/saml/login, should redirect to IdP

**4. RBAC Configuration**
- Role matrix table (viewer/analyst/admin permissions)
- Default role for SAML users: viewer (configurable via RBAC_DEFAULT_ROLE env var)
- Admin API: how to assign roles via PATCH /api/v1/admin/users/{id}/role
- First admin setup: manually UPDATE users SET role='admin' WHERE email='it-admin@company.com' via psql

**5. Audit Log Configuration**
- Audit log is enabled by default; no configuration needed
- Retention: default 365 days (configurable via AUDIT_RETENTION_DAYS)
- Export: GET /api/v1/admin/audit-log?start=...&end=...&format=csv
- SIEM integration: schedule CSV export to Splunk/ServiceNow/Elastic via cron or webhook consumer
- Database query: direct Postgres access for custom reports

**6. Air-Gapped Installation**
- Step-by-step for fully offline deployment:
  1. On build machine (internet access): run export-images.sh
  2. Transfer cadverify-enterprise/ directory to air-gapped server via USB/SCP
  3. On air-gapped server: run load-images.sh
  4. Copy .env.example to .env, configure
  5. docker compose up -d
  6. Run init-db.sh for migrations
  7. Verify with health-check.sh
- Troubleshooting: "image not found" = load-images.sh not run; "connection refused" = check .env for correct hostnames

**7. Kubernetes / Helm Deployment**
- Install chart: `helm install cadverify charts/cadverify -f values-enterprise.yaml`
- Values walkthrough: replica counts, resource limits, persistence sizes, SAML config, ingress
- Air-gapped Kubernetes: load images with `ctr -n k8s.io images import`, set pullPolicy: Never
- TLS: configure ingress.tls with cert-manager or pre-provisioned secret
- Alembic migration: runs automatically as Helm hook on install/upgrade

**8. Upgrading**
- Docker: pull new images (or load new tars), run init-db.sh for new migrations, docker compose up -d
- Kubernetes: `helm upgrade cadverify charts/cadverify -f values-enterprise.yaml` (migration job runs automatically)
- Rollback: `helm rollback cadverify` or restore previous image tars
- Data is preserved in named volumes/PVCs across upgrades

**9. Troubleshooting**
- Common issues table:
  | Symptom | Cause | Fix |
  |---------|-------|-----|
  | SAML login fails with "Invalid response" | Clock skew >5min | Sync NTP on both SP and IdP servers |
  | SAML login fails with "Signature validation failed" | Wrong IdP metadata | Re-download idp_metadata.xml from IdP |
  | 403 on analysis endpoint | User has viewer role | Admin: PATCH user role to analyst |
  | Backend fails to start | Missing env vars | Check .env against .env.example |
  | "Image not found" on docker compose up | Images not loaded | Run scripts/load-images.sh |
  | Alembic migration fails | DB not ready | Wait for postgres healthcheck, retry init-db.sh |
- Log locations: `docker compose logs backend`, `docker compose logs worker`
- Health endpoint: GET /health returns {"db": "ok", "redis": "ok"} when healthy
